## Challenges and Growth Areas

- Multiple stories exposed recurring reliability concerns in modal initialization and event handling
- Third-party embed behavior needed explicit hardening (origin/source validation, fallback behavior)
- Source-level smoke tests are strong, but runtime browser behavior coverage remains the main quality gap
- n8n workflow artifacts are documented and versioned, but full environment validation still depends on staged credentials and endpoint checks

## Cross-Epic Lessons Learned

1. Reliability around user-triggered modal flows must be designed as a first-class concern, not a final hardening step.
2. Third-party integration trust boundaries should be explicit from the first implementation pass.
3. Static/source-level tests are excellent for regression speed, but high-risk user journeys need browser-executed assertions.
4. Keeping CI gates strict (lint, build, Lighthouse, tests) prevented quality drift as scope expanded.
5. Operationally editable settings (Supabase global settings) reduce business bottlenecks without sacrificing stability when fallback strategy is implemented.
